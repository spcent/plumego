# Docs Home

<div class="docs-home-intro">
  <p class="docs-home-intro__lead">
    Plumego is a stdlib-first Go web toolkit for explicit HTTP services. The shortest way into the repository is to start with the canonical reference path, then move
    outward into boundaries and extension families only when you need them.
  </p>
  <p class="docs-home-intro__support">
    This docs home is meant to feel like an entry portal, not a marketing splash. Start with the recommended path, build the concept layer first, inspect the synced
    repository facts, and only then branch into the module surface.
  </p>
</div>
