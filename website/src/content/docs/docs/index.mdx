# Docs Home

Plumego is a stdlib-first Go web toolkit for explicit HTTP services. Use this page as the implementation handoff: run the reference path, inspect the canonical service shape, then move into boundaries only when the question becomes ownership.

## First successful path

| Step | Command | Expected signal | Next page |
| --- | --- | --- | --- |
| Start the reference service | `go run ./reference/standard-service` | The service listens on `:8080` | [Getting Started](/docs/getting-started) |
| Check liveness | `curl http://localhost:8080/healthz` | JSON response with `data.status` set to `ok` | [Reference App](/docs/reference-app) |
| Check a routed API handler | `curl 'http://localhost:8080/api/v1/greet?name=Alice'` | `{"data":{"message":"hello, Alice"}}` | [Request Flow](/docs/concepts/request-flow) |

## Stable surface at a glance

| Surface | Use it for | Do not use it for |
| --- | --- | --- |
| Stable roots | Kernel, routing, contracts, middleware, security, storage, health, logging, and metrics basics | Optional product capabilities or protocol-specific feature work |
| `reference/standard-service` | Canonical service shape and runnable examples | Repository ownership decisions |
| `x/*` families | Optional capability work that should not expand stable roots | The default learning path for every service |
| `docs`, `specs`, `tasks` | Human intent, machine-readable rules, and execution scope | Runtime package APIs |
