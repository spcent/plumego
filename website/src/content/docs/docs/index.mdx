# Docs Home

<div class="docs-home-intro">
  <p class="docs-home-intro__lead">
    Plumego is a stdlib-first Go web toolkit for explicit HTTP services. The shortest way into the repository is to start with the canonical reference path, then move
    outward into boundaries and extension families only when you need them.
  </p>
  <p class="docs-home-intro__support">
    If you want the fastest proof, run the reference service first. If you already know it runs, use the reading path below to decide whether the next question is
    service shape, repository boundaries, or an extension family.
  </p>
</div>

<div class="docs-home-coverage not-content">
  <p class="signal-list__kicker">first run</p>
  <h2>Fastest successful path</h2>
  <p>From the repository root, start the canonical service and verify the two endpoints that prove the HTTP path is live.</p>
  <pre><code>go run ./reference/standard-service
curl http://localhost:8080/healthz
curl http://localhost:8080/api/v1/greet?name=Alice</code></pre>
  <p>After that, continue to Getting Started for the small inspection checklist or Reference App for the file-by-file service shape.</p>
</div>
