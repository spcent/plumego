# Real-Time with WebSockets

This guide shows how to add WebSocket support to a Plumego service using `x/websocket`. You will upgrade HTTP connections, join clients to named rooms, and broadcast messages to connected clients.

`x/websocket` is experimental. See [Release Posture](/docs/release-posture) and [Extension Maturity](/docs/concepts/extension-maturity) before depending on it in production.

Boundary rationale is in the [x/websocket Primer](/docs/modules/x-websocket).

## Step 1 â€” Create the Server

`x/websocket.New` returns a `*Server` that owns a `Hub`, an auth handler, and its own route registration:

```go
import (
    "os"

    "github.com/spcent/plumego/log"
    "github.com/spcent/plumego/x/websocket"
)

cfg := websocket.DefaultWebSocketConfig()
cfg.WSRoutePath      = "/ws"          // WebSocket upgrade endpoint
cfg.Secret           = []byte(os.Getenv("WS_SECRET"))
cfg.AllowedOrigins   = []string{"https://app.example.com"}
cfg.MaxRoomRegistrations     = 1000
cfg.MaxRoomConnections = 100

logger := log.NewLogger()
ws, err := websocket.New(cfg, false, logger)
if err != nil {
    return err
}
```

`DefaultWebSocketConfig()` sets worker count, queue sizes, and timeouts. Keep
`WS_SECRET` at 32 bytes or longer when token auth is required, set
`AllowedOrigins` for browser clients, and leave `BroadcastEnabled` off unless
you explicitly need the admin broadcast route.

## Step 2 â€” Register routes

Mount the WebSocket endpoint on your application router:

```go
import "github.com/spcent/plumego/router"

r := router.NewRouter()
if err := ws.RegisterRoutes(r); err != nil {
    return err
}
```

`RegisterRoutes` mounts the upgrade handler at `cfg.WSRoutePath`. Clients connect to that path using the standard WebSocket handshake.

## Step 3 â€” Broadcast to a room

Retrieve the `Hub` and send messages from any goroutine:

```go
hub := ws.Hub()

// Broadcast a text message to all connections in "chat:general"
hub.BroadcastRoom("chat:general", websocket.OpcodeText, []byte(`{"type":"message","text":"Hello"}`))

// Broadcast to every connected client across all rooms
hub.BroadcastAll(websocket.OpcodeText, []byte(`{"type":"ping"}`))
```

Use `websocket.OpcodeText` for JSON or UTF-8 strings and
`websocket.OpcodeBinary` for binary frames. `BroadcastRoom` and `BroadcastAll`
are fire-and-forget helpers; use `TryBroadcastRoom` or `TryBroadcastAll` when
callers need sent and dropped counts.

## Step 4 â€” Custom handler with rooms and auth

For full control â€” custom room names, bearer-token auth, connection metadata â€” call `ServeWSWithConfig` or `ServeRoomFanoutWS` directly from your own handler:

```go
import (
    "net/http"
    "time"
    "github.com/spcent/plumego/x/websocket"
)

type ChatHandler struct {
    hub       *websocket.Hub
    roomAuth  websocket.RoomAuthorizer
    tokenAuth websocket.TokenAuthenticator
}

func (h *ChatHandler) ServeHTTP(w http.ResponseWriter, r *http.Request) {
    websocket.ServeRoomFanoutWS(w, r, websocket.ServerConfig{
        Hub:          h.hub,
        RoomAuth:     h.roomAuth,
        TokenAuth:    h.tokenAuth,
        QueueSize:    256,               // send queue size per connection
        SendTimeout:  5*time.Second,     // send timeout
        SendBehavior: websocket.SendDrop, // drop message if queue full
    })
}
```

Wire the auth and hub at startup:

```go
roomAuth  := websocket.NewSimpleRoomAuth()
tokenAuth, err := websocket.NewHS256TokenAuth([]byte(os.Getenv("WS_SECRET")))
if err != nil {
    return err
}
hub       := websocket.NewHub(4, 512)  // 4 workers, 512 job queue

r.Get("/ws/chat", &ChatHandler{hub: hub, roomAuth: roomAuth, tokenAuth: tokenAuth})
```

`NewHS256TokenAuth` is a lightweight HS256 verifier. It validates the
signature and optional integer `exp` claim, but does not enforce issuer,
audience, `nbf`, or `iat`. Use `security/jwt` or a custom
`websocket.TokenAuthenticator` for full JWT/OIDC policy.

### Room passwords

Protect specific rooms with a per-room password:

```go
roomAuth.SetRoomPassword("vip-room", "s3cr3t")
```

Clients include the password in the `X-Room-Password` header during the upgrade request.
Room passwords in URL query parameters are rejected. Bearer tokens should be
sent in `Authorization: Bearer ...`; query tokens are disabled unless
`AllowQueryToken` is explicitly set.

## Step 5 â€” Inspect hub state

```go
// Number of active connections in a room
count := hub.GetRoomCount("chat:general")

// Room registrations across all rooms
total := hub.GetRoomRegistrationCount()

// List all room names
rooms := hub.GetRooms()
```

Room registrations are not unique connections. If one connection joins three
rooms, it counts as three registrations.

## Admin broadcast route

The admin broadcast route is disabled by default. To expose it, set
`BroadcastEnabled` and provide a separate `BroadcastSecret` of at least 32
bytes. Clients must send that value in the `Authorization: Bearer ...` header;
the endpoint does not read secrets from URLs.

## Step 6 â€” Graceful shutdown

Shut down the WebSocket server as part of your application shutdown sequence:

```go
ctx, cancel := context.WithTimeout(context.Background(), 10*time.Second)
defer cancel()

if err := ws.Shutdown(ctx); err != nil {
    logger.Error("websocket shutdown error", log.Fields{"err": err.Error()})
}
```

`Shutdown` stops accepting new joins, closes open TCP connections, and drains
queued send jobs for a bounded period before returning. It does not wait for a
full WebSocket close handshake with each peer.

## SendBehavior options

| Constant | Effect when client queue is full |
| --- | --- |
| `websocket.SendBlock` | Block until space is available |
| `websocket.SendDrop` | Silently drop the message |
| `websocket.SendClose` | Close the connection |

Use `SendDrop` for real-time feeds where a slow client should not stall the broadcaster. Use `SendBlock` when message delivery is critical and bounded latency is acceptable.
