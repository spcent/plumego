# 文档首页

Plumego 是一个 stdlib-first 的 Go Web 工具包，强调显式 HTTP 服务结构。可以把这页当成实现阶段的交接页：先跑参考路径，再看 canonical 服务形态，只有当问题变成“归谁负责”时才进入仓库边界。

## 最短成功路径

| 步骤 | 命令 | 预期信号 | 下一页 |
| --- | --- | --- | --- |
| 启动参考服务 | `go run ./reference/standard-service` | 服务监听在 `:8080` | [开始使用](/zh/docs/getting-started) |
| 检查存活 | `curl http://localhost:8080/healthz` | JSON 响应里 `data.status` 为 `ok` | [参考应用](/zh/docs/reference-app) |
| 检查已注册 API handler | `curl 'http://localhost:8080/api/v1/greet?name=Alice'` | `{"data":{"message":"hello, Alice"}}` | [请求流程](/zh/docs/concepts/request-flow) |

## 阅读地图

## 稳定表面速查

| 表面 | 适合用于 | 不适合用于 |
| --- | --- | --- |
| 稳定根 | 内核、路由、合约、中间件、安全、存储、健康、日志和指标基础能力 | 可选产品能力或协议型功能工作 |
| `reference/standard-service` | canonical 服务形态和可运行示例 | 仓库归属判断 |
| `x/*` 家族 | 不应扩张稳定根的可选能力工作 | 每个服务都必须学习的默认路径 |
| `docs`、`specs`、`tasks` | 人类意图、机器可读规则和执行范围 | 运行时包 API |
