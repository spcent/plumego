# 实时通信 WebSocket

本指南展示如何使用 `x/websocket` 为 Plumego 服务添加 WebSocket 支持。你将升级 HTTP 连接、将客户端加入命名房间，并向已连接客户端广播消息。

`x/websocket` 为 experimental 状态。在生产中使用前请参阅[发布策略](/zh/docs/release-posture)和[扩展成熟度](/zh/docs/concepts/extension-maturity)。

边界说明见 [x/websocket Primer](/zh/docs/modules/x-websocket)。

## 第一步 — 创建 Server

`x/websocket.New` 返回一个 `*Server`，它包含 Hub、认证处理器和路由注册：

```go
import (
    "os"

    "github.com/spcent/plumego/log"
    "github.com/spcent/plumego/x/websocket"
)

cfg := websocket.DefaultWebSocketConfig()
cfg.WSRoutePath        = "/ws"          // WebSocket 升级端点
cfg.Secret             = []byte(os.Getenv("WS_SECRET"))
cfg.AllowedOrigins     = []string{"https://app.example.com"}
cfg.MaxRoomRegistrations     = 1000
cfg.MaxRoomConnections = 100

logger := log.NewLogger()
ws, err := websocket.New(cfg, false, logger)
if err != nil {
    return err
}
```

`DefaultWebSocketConfig()` 设置 worker 数量、队列大小和超时。需要
token auth 时，`WS_SECRET` 必须至少 32 字节；面向浏览器客户端时应设置
`AllowedOrigins`；除非明确需要 admin broadcast 路由，否则保持
`BroadcastEnabled` 关闭。

## 第二步 — 注册路由

在应用路由器上挂载 WebSocket 端点：

```go
import "github.com/spcent/plumego/router"

r := router.NewRouter()
if err := ws.RegisterRoutes(r); err != nil {
    return err
}
```

`RegisterRoutes` 在 `cfg.WSRoutePath` 挂载升级 handler。客户端通过标准 WebSocket 握手连接到该路径。

## 第三步 — 向房间广播

获取 Hub 并从任意 goroutine 发送消息：

```go
hub := ws.Hub()

// 向 "chat:general" 房间的所有连接广播文本消息
hub.BroadcastRoom("chat:general", websocket.OpcodeText, []byte(`{"type":"message","text":"Hello"}`))

// 向所有房间的所有连接广播
hub.BroadcastAll(websocket.OpcodeText, []byte(`{"type":"ping"}`))
```

对 JSON 或 UTF-8 字符串使用 `websocket.OpcodeText`，对二进制帧使用
`websocket.OpcodeBinary`。`BroadcastRoom` 和 `BroadcastAll` 是
fire-and-forget 助手；如果调用方需要 sent/dropped 计数，请使用
`TryBroadcastRoom` 或 `TryBroadcastAll`。

## 第四步 — 带房间和认证的自定义 handler

如需完全控制 — 自定义房间名称、bearer token 认证、连接元数据 — 直接在自己的 handler 中调用 `ServeWSWithConfig` 或 `ServeRoomFanoutWS`：

```go
import (
    "net/http"
    "time"
    "github.com/spcent/plumego/x/websocket"
)

type ChatHandler struct {
    hub       *websocket.Hub
    roomAuth  websocket.RoomAuthorizer
    tokenAuth websocket.TokenAuthenticator
}

func (h *ChatHandler) ServeHTTP(w http.ResponseWriter, r *http.Request) {
    websocket.ServeRoomFanoutWS(w, r, websocket.ServerConfig{
        Hub:          h.hub,
        RoomAuth:     h.roomAuth,
        TokenAuth:    h.tokenAuth,
        QueueSize:    256,               // 每个连接的发送队列大小
        SendTimeout:  5*time.Second,     // 发送超时
        SendBehavior: websocket.SendDrop, // 队列满时丢弃消息
    })
}
```

在启动时 wiring auth 和 hub：

```go
roomAuth  := websocket.NewSimpleRoomAuth()
tokenAuth, err := websocket.NewHS256TokenAuth([]byte(os.Getenv("WS_SECRET")))
if err != nil {
    return err
}
hub       := websocket.NewHub(4, 512)  // 4 个 worker，512 个任务队列

r.Get("/ws/chat", &ChatHandler{hub: hub, roomAuth: roomAuth, tokenAuth: tokenAuth})
```

`NewHS256TokenAuth` 是轻量 HS256 verifier。它校验签名和可选的整数
`exp` claim，但不负责 issuer、audience、`nbf` 或 `iat` 策略。完整
JWT/OIDC 策略请使用 `security/jwt` 或自定义 `websocket.TokenAuthenticator`。

### 房间密码

用每个房间的密码保护特定房间：

```go
roomAuth.SetRoomPassword("vip-room", "s3cr3t")
```

客户端在升级请求时通过 `X-Room-Password` header 提供密码。URL query
中的 room password 会被拒绝。Bearer token 应通过
`Authorization: Bearer ...` 发送；除非显式设置 `AllowQueryToken`，否则
query token 会被拒绝。

## 第五步 — 查看 Hub 状态

```go
// 某个房间的活跃连接数
count := hub.GetRoomCount("chat:general")

// 所有房间的 room registration 数
total := hub.GetRoomRegistrationCount()

// 列出所有房间名称
rooms := hub.GetRooms()
```

Room registrations 不是唯一连接数。同一个连接加入三个房间时，会计为
三个 registrations。

## Admin broadcast 路由

Admin broadcast 路由默认关闭。如需暴露它，设置 `BroadcastEnabled`，并
提供独立的 `BroadcastSecret`，长度至少 32 字节。客户端必须通过
`Authorization: Bearer ...` header 发送该值；该端点不会从 URL 读取
secret。

## 第六步 — 优雅关闭

在应用关闭序列中关闭 WebSocket Server：

```go
ctx, cancel := context.WithTimeout(context.Background(), 10*time.Second)
defer cancel()

if err := ws.Shutdown(ctx); err != nil {
    logger.Error("websocket shutdown error", log.Fields{"err": err.Error()})
}
```

`Shutdown` 停止接受新的 join，关闭已打开的 TCP 连接，并在有界时间内
排空已排队的发送任务后返回。它不会等待每个 peer 完整完成 WebSocket
close handshake。

## SendBehavior 选项

| 常量 | 客户端队列满时的行为 |
| --- | --- |
| `websocket.SendBlock` | 阻塞直到有空间 |
| `websocket.SendDrop` | 静默丢弃消息 |
| `websocket.SendClose` | 关闭连接 |

对于慢客户端不应阻塞广播方的实时信息流，使用 `SendDrop`。当消息投递是关键需求且延迟有界时，使用 `SendBlock`。
