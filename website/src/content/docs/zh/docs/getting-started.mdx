# 开始使用

## 无需安装，直接尝试

在浏览器中打开参考服务 — 无需本地安装 Go：

[![在 GitHub Codespaces 中打开](https://github.com/codespaces/badge.svg)](https://codespaces.new/spcent/plumego?quickstart=1)

Codespace 内已预装 Go 工具链。运行：

```bash
go run ./reference/standard-service
```

然后打开转发的端口并验证：

```bash
curl http://localhost:8080/healthz
# {"data":{"status":"ok","service":"plumego-reference","check":"liveness"}}
```

## 最快起步 — 直接添加到任意 Go 项目

环境要求：Go 1.24+ 和一个现有的 Go module（`go.mod`）。

```bash
go mod init myservice        # 已有 module 可跳过
go get github.com/spcent/plumego@latest
```

创建 `main.go`：

```go
package main

import (
    "log"
    "net/http"

    "github.com/spcent/plumego/contract"
    "github.com/spcent/plumego/core"
    plog "github.com/spcent/plumego/log"
)

func main() {
    app := core.New(core.DefaultConfig(),
        core.AppDependencies{Logger: plog.NewLogger()})

    _ = app.Get("/ping", http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        _ = contract.WriteResponse(w, r, http.StatusOK,
            map[string]string{"status": "ok"}, nil)
    }))

    if err := app.Prepare(); err != nil {
        log.Fatal(err)
    }
    srv, _ := app.Server()
    log.Println("listening on :8080")
    srv.ListenAndServe()
}
```

运行并验证：

```bash
go run .
curl http://localhost:8080/ping
# {"data":{"status":"ok"}}
```

### 这段代码证明了什么

| 信号 | 说明 |
| --- | --- |
| `contract.WriteResponse` 包装了 map | 成功响应使用稳定传输合约 |
| `app.Get(path, http.HandlerFunc(...))` | 路由注册显式，没有隐藏的 `init()` |
| `app.Prepare()` 然后 `app.Server()` | 生命周期分两步显式调用，不是框架托管的后台启动 |

## 想看完整的 canonical 服务形态？

克隆参考应用，看完整目录布局——包括配置加载、健康检查端点、handler 分层和 constructor-based 依赖注入。

```bash
git clone https://github.com/spcent/plumego
go run ./reference/standard-service
```

验证：

```bash
curl http://localhost:8080/healthz
# {"data":{"status":"ok","service":"plumego-reference","check":"liveness"}}

curl 'http://localhost:8080/api/v1/greet?name=Alice'
# {"data":{"message":"hello, Alice"}}
```

四个文件覆盖了完整的服务形态：

| 文件 | 该看什么 |
| --- | --- |
| `reference/standard-service/main.go` | 启动顺序 |
| `reference/standard-service/internal/app/app.go` | 显式依赖注入 |
| `reference/standard-service/internal/app/routes.go` | 路由归属集中在一处 |
| `reference/standard-service/internal/handler/api.go` | Handler 形态：标准库兼容 |
