# M-019: Module Ecosystem Foundation

- **Branch:** `milestone/M-019-module-ecosystem`
- **Depends on:** M-017
- **Parallel OK:** yes

## Goal

Establish the foundational contract and tooling for community-authored plumego
extensions: a machine-readable community extension schema, a `plumego add`
command that validates schema compliance, and an authoring guide that gives
third-party developers a clear, checkable contract for publishing compatible
extensions.

## Architecture Decisions

- The community extension schema lives in specs/community-extension.schema.yaml
  following the existing module-manifest.schema.yaml pattern.
- `plumego add <module-path>` downloads the module, validates its
  community-extension.yaml against the schema, and adds it to go.mod; it does
  NOT auto-wire routes or inject globals.
- The schema enforces: no stable-root imports, explicit module.yaml with status
  field, at least one test command declared, no init() side effects (checked by
  static analysis), handler shape compliance (func(http.ResponseWriter, *http.Request)).
- docs/EXTENSION_AUTHORING.md is the canonical guide; it references the schema
  and the add command.
- x/rpc is used as the first example community extension in the guide (it
  already follows all constraints).
