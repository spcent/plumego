# M-016: Event-Driven Reference Architecture

- **Branch:** `milestone/M-016-event-driven-reference`
- **Depends on:** M-010
- **Parallel OK:** yes

## Goal

Ship reference/with-events demonstrating the outbox pattern, durable consumer,
retry loop, and webhook delivery using x/messaging primitives, giving teams a
concrete end-to-end reference for event-driven services built on plumego.

## Architecture Decisions

- reference/with-events is a self-contained Go module that imports only stable
  roots and x/messaging; no new framework features are introduced.
- The reference uses an in-process pubsub backend for the demo; no external
  broker is required to run locally.
- Outbox pattern is demonstrated at the application layer (not a framework
  feature): handler writes event to outbox table inside a DB transaction, a
  goroutine relays to pubsub.
- Webhook delivery uses x/messaging/webhook with an exponential-backoff retry;
  the target URL is configurable via environment variable.
- The reference must compile and run with `go run ./...` with no environment
  variables beyond optional ones.
