# M-017: gRPC Support

- **Branch:** `milestone/M-017-grpc-support`
- **Depends on:** M-014
- **Parallel OK:** yes

## Goal

Ship x/rpc with server, client, and gateway sub-packages that integrate gRPC
lifecycle with core.App and connect to x/gateway for HTTP/JSON transcoding,
giving plumego services a first-class gRPC story.

## Architecture Decisions

- x/rpc has its own go.mod and depends on google.golang.org/grpc; it is never
  imported by the main module or stable roots.
- x/rpc/server wraps grpc.Server lifecycle (Start/GracefulStop) with the same
  context-cancellation contract as core.App.Shutdown.
- x/rpc/client provides a connection pool with interceptors for structured
  logging, distributed tracing (via x/observability), and retry.
- x/rpc/gateway bridges x/gateway's reverse-proxy with gRPC-HTTP transcoding
  using grpc-gateway; it is a separate sub-package to keep the dependency opt-in.
- x/rpc/module.yaml status = experimental on first release.
