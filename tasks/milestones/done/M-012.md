# M-012: Input Validation Bridge

- **Branch:** `milestone/M-012-input-validation-bridge`
- **Depends on:** M-008
- **Parallel OK:** yes

## Goal

Ship x/validate with a Validator interface and generic Bind[T] helper that
eliminates the most common binding-and-validation boilerplate, with an
optional x/validate/playground sub-package adapting go-playground/validator.

## Architecture Decisions

- x/validate defines only the Validator interface and Bind[T]; it has zero
  external dependencies beyond the main module stable roots.
- The go-playground/validator dependency lives exclusively in
  x/validate/playground/ which has its own go.mod; this preserves
  stdlib-only status for the main module.
- Bind[T] uses Go generics (1.21+); JSON decode uses json.NewDecoder(r.Body).
- Validation errors are returned as contract.APIError via contract.WriteError;
  no new error envelope types.
- x/validate must not import x/* packages other than contract.

## Out of Scope

- Do not add go-playground/validator to the main module go.mod.
- Do not add validation logic to stable root packages.

- Do not add validation middleware; keep validation explicit at call sites.
