# M-005: v1 Release Execution

- **Branch:** `milestone/M-005-v1-release-execution`
- **Depends on:** M-001, M-002, M-003, M-004
- **Parallel OK:** no

## Goal

Publish a verifiable `v1.0.0` release only after the release candidate, stable
root freeze, CLI onboarding, extension maturity, and final release evidence all
close without unresolved P0/P1 blockers.

## Architecture Decisions

- Treat `v1.0.0-rc.1` as the next release gate; do not tag final `v1.0.0`
  until rc observation is recorded.
- Do not promote experimental extensions for v1 without complete release refs,
  release-backed snapshots, and owner sign-off.
- Do not change runtime behavior during release execution unless a gate failure
  produces a bounded blocker card.
- Keep stable roots dependency-free beyond the standard library and preserve the
  existing public API unless an explicit symbol-change card authorizes a change.
- Release notes must describe implemented behavior and recorded evidence only.
