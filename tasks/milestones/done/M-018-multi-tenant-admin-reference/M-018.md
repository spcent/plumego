# M-018: Multi-Tenant Admin Reference

- **Branch:** `milestone/M-018-tenant-admin-reference`
- **Depends on:** M-010
- **Parallel OK:** yes

## Goal

Ship reference/with-tenant-admin demonstrating tenant lifecycle management,
quota administration, and usage recording using x/tenant primitives, giving
SaaS teams a concrete reference for the administrative plane of a multi-tenant
platform.

## Architecture Decisions

- reference/with-tenant-admin is a self-contained Go module importing only
  stable roots and x/tenant; no new framework features are introduced.
- Admin API uses the same handler shape as all other reference apps:
  func(http.ResponseWriter, *http.Request).
- Tenant CRUD (create, read, suspend, delete) is demonstrated at the application
  layer; x/tenant provides resolution and policy enforcement, not CRUD logic.
- Quota management shows read-quota, set-quota, and reset-quota endpoints using
  x/tenant/quota contracts.
- Usage recording uses an in-memory store for the demo; the adapter interface
  makes it swappable for a real DB.
- All admin endpoints require an authorization header check (fail closed).
