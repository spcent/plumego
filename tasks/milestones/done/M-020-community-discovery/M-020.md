# M-020: Community Discovery & Documentation Overhaul

- **Branch:** `milestone/M-020-community-discovery`
- **Depends on:** M-008
- **Parallel OK:** yes

---

## Goal

Make plumego self-explanatory to a Go developer arriving for the first time — via `pkg.go.dev`, the README, or a web search — by overhauling package-level doc comments across all stable roots and rewriting the README with a 5-minute quick-start and a Go-ecosystem comparison table.

---

## Architecture Decisions

- No code changes. This milestone is documentation-only.
- Package-level doc comments must follow the `pkg.go.dev` contract: first sentence is the package summary (no "Package X" prefix needed beyond the convention), body contains a minimal runnable usage example using `Example` function format where appropriate.
- README structure: Quick Start (≤30 lines of Go, no explanation of internals) → Why plumego (3 bullet points) → stdlib ServeMux vs. plumego table → Package overview → Getting Help.
- README_CN.md mirrors README.md structure. Translations must be accurate, not machine-translated without review.
- The stdlib comparison table is the highest-value single artifact in this milestone: it answers the Go 1.22+ question ("why not just use the new ServeMux?") definitively.
- Do not add doc comments that describe implementation details; document observable behavior and usage patterns only.
- Do not add emojis, badges beyond existing CI badge, or marketing language.
