# M-015: Database Adapters

- **Branch:** `milestone/M-015-database-adapters`
- **Depends on:** M-009
- **Parallel OK:** yes

## Goal

Ship x/data/pgx, x/data/sqlx, and x/data/migrate as dependency-free adapter
contracts and migration helpers. Concrete database drivers remain caller-owned,
while Plumego provides explicit query, transaction, and migration surfaces that
are tested offline without a live database.

## Architecture Decisions

- The main module remains dependency-free beyond the standard library; pgx,
  sqlx, goose, and sqlite drivers are not imported by the main module.
- Adapter packages expose driver-shaped `Querier` and `Transactor` contracts
  inside `x/data`; stable `store/db` interfaces are not modified.
- Offline tests use local mocks and stdlib `database/sql` shapes; no live
  PostgreSQL or SQLite driver is required for CI.
- The plumego migrate command extension lives in cmd/plumego/commands/migrate.go
  (which already exists) and uses `database/sql` with caller-registered drivers.
- x/data/pgx and x/data/sqlx module.yaml status = experimental on first release.
