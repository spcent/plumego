# M-003: Extension Evidence Pipeline

- **Branch:** `milestone/M-003-extension-evidence`
- **Depends on:** M-002
- **Parallel OK:** no

## Goal

In 4 weeks, build a tiered evidence pipeline for `x/*`, complete `x/rest` beta evidence first, then establish reusable assessment templates for `x/tenant`, `x/observability`, and `x/gateway`.

## Architecture Decisions

- No extension without complete beta evidence may be labeled as stable or GA.
- `x/*` changes are accepted only through observable-behavior and boundary tests; not by feature-bulking.
- Extensions must not couple back into stable roots; any boundary violation stops and is rolled back.

## Tasks

### Phase 1 — Evidence Baseline (sequential)

1. Create a standard evidence template for `x/*` (stability, compatibility, error semantics, boundary dependencies, minimal examples).
2. Define beta criteria for `x/rest` (minimum API surface, compatibility policy, regression scripts).

### Phase 2 — Implement (parallel)

- [ ] `x/rest`: Add API snapshot coverage and error-model consistency tests; update module README and module YAML entry points.
- [ ] `x/tenant`: Clarify tenant/session lifecycle and isolation boundaries, and add critical isolation tests.
- [ ] `x/observability`: Add minimal regression coverage for sampling, export, and error-path observability behavior.
- [ ] `x/gateway`: Record adapter-readiness conditions in the todo docs and avoid new release commitments.
- [ ] `specs/extension-maturity.yaml`: Add milestone-specific state and evidence references.

### Phase 3 — Validate (sequential)

1. Produce module-level evidence artifacts at PR scope (e.g. under `docs/extension-evidence/`).
2. Sync extension status in `docs/ROADMAP.md` and remove unimplemented commitments.
3. Complete milestone-level verification and rollback conditions.

### Weekly checkpoints (verifiable)

1. Week 1: Finalize the evidence template and baseline criteria, then create the `x/rest` evidence matrix.
2. Week 2: Deliver `x/rest` API snapshot and error-path comparison results.
3. Week 3: Deliver boundary tests and documentation alignment for `x/tenant` and `x/observability`.
4. Week 4: Deliver `x/gateway` conditional state and a full milestone summary; carry over blockers to next milestone.
