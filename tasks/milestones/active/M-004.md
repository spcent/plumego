# M-004: Stable Root Cleanup Freeze

- **Branch:** `milestone/M-004-stable-root-cleanup`
- **Depends on:** M-002
- **Parallel OK:** no

## Goal

In 4 weeks, clean up stale, duplicated, and inconsistent stable-root code while preserving the v1 public API freeze and net/http compatibility.

## Architecture Decisions

- Stable root public APIs must not change unless an active card explicitly uses the exported-symbol change protocol.
- Existing v1 compatibility aliases and lenient constructors stay unless their removal is explicitly approved and all callers are migrated in the same change.
- Stable roots must remain dependency-free beyond the standard library and must not import `x/*`.
- Cleanup work must prefer mechanical splits, internal helper convergence, and documentation freeze over behavioral rewrites.
- Response and error behavior must keep one canonical structured path per layer; visible response-shape changes require tests and docs sync.
