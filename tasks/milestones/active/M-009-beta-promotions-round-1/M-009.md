# M-009: Beta Promotions Round 1

- **Branch:** `milestone/M-009-beta-promotions-r1`
- **Depends on:** M-008
- **Parallel OK:** yes

## Goal

Promote x/ai stable-tier subpackages (provider, session, tool, streaming),
x/tenant, x/gateway/discovery core-static, and x/data/file plus
x/data/idempotency from experimental to beta status, using the two-release
evidence now available after v1.1.0.

## Architecture Decisions

- Each promotion requires: two release refs, release-to-release API snapshots
  with no exported-symbol changes, and owner sign-off recorded in
  specs/extension-beta-evidence.yaml.
- Update the module.yaml status field and docs/EXTENSION_MATURITY.md for each
  promoted surface in the same commit as the evidence ledger.
- Do not promote root x/ai; promote only the four stable-tier subpackages.
- Do not alter runtime behavior of any promoted package.
- x/ai/provider, x/ai/session, x/ai/tool, x/ai/streaming are promoted
  individually as separate evidence entries.
