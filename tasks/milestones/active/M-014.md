# M-014: OpenAPI 3.1 Generation

- **Branch:** `milestone/M-014-openapi-generation`
- **Depends on:** M-009
- **Parallel OK:** yes

## Goal

Ship x/openapi with a route-driven OpenAPI 3.1 document generator and integrate
it into the plumego CLI as `plumego generate spec`, turning the framework's
explicit route registration into a first-class API documentation capability.

## Architecture Decisions

- x/openapi depends only on the main module stable roots (router, contract) and
  stdlib; no external OpenAPI library dependency.
- Route hints (x/openapi.Op) are optional; routes without hints still appear
  in the spec with method, path, and a default 200 response.
- The generator reads router.Routes() directly; it does not use reflection on
  handler functions.
- The CLI command `plumego generate spec` writes to stdout or --output path;
  format is JSON by default, YAML with --format yaml flag.
- x/openapi/module.yaml sets status = experimental on first release.
