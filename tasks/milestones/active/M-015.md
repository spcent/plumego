# M-015: Database Adapters

- **Branch:** `milestone/M-015-database-adapters`
- **Depends on:** M-009
- **Parallel OK:** yes

## Goal

Ship x/data/pgx (pgx v5 adapter), x/data/sqlx (sqlx adapter), and x/data/migrate
(goose-backed migration runner) as independently versioned sub-packages, each
implementing the store/db contracts and tested offline without a live database.

## Architecture Decisions

- Each sub-package has its own go.mod under x/data/pgx/, x/data/sqlx/, and
  x/data/migrate/; they are not part of the main module.
- Each adapter implements the Querier and Transactor interfaces from store/db;
  no new interface types are introduced in stable roots.
- Offline tests use sqlmock or an in-memory SQLite driver accessed via
  database/sql; no live PostgreSQL required for CI.
- The plumego migrate command extension lives in cmd/plumego/commands/migrate.go
  (which already exists) and calls x/data/migrate via a plugin hook, not a
  direct import in the main CLI module.
- x/data/pgx and x/data/sqlx module.yaml status = experimental on first release.
