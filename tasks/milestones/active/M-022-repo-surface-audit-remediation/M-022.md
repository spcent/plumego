## Goal

The repo's machine-readable API surface, docs, and extension architecture are aligned with current code, and the verified duplicate or placeholder surfaces from the 2026-05 audit have an explicit remediation path.

---

## Architecture Decisions

- Only findings verified against current `main` are in scope; corrected or overstated audit claims stay out of implementation cards.
- Land manifest and docs surface sync before any behavior-changing refactor.
- Prefer convergence onto shared `x/resilience/*` primitives and stable `metrics` contracts; do not add new parallel resilience or metrics stacks.
- Avoid stable exported symbol renames unless the card includes caller migration and same-change cleanup.
- Compatibility aliases and unsupported placeholder fields must either gain an explicit removal window or be removed with caller migration.
