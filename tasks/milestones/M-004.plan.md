# Plan for M-004: Stable Root Cleanup Freeze

Milestone: `M-004`
Objective: Clean up stale, duplicated, and inconsistent stable-root implementation details while preserving the v1 public API freeze.
Constraints: no new dependencies, no `x/*` imports from stable roots, no stable public API changes without an explicit symbol-change card, and no release tagging.
Affected Modules: `contract`, `router`, `middleware`, `security`, `store`, `log`, `metrics`, `health`, `core`

## Phase Map

- Phase 1: Freeze stable-root compatibility decisions and close existing contract hardening cards.
- Phase 2: Normalize router and middleware transport behavior where behavior is already intended to be canonical.
- Phase 3: Freeze security/store compatibility guidance and mechanically reduce store file size.
- Phase 4: Mechanically split the stable logging backend.
- Phase 5: Run stable-root gates and record release-readiness evidence.

## Parallel Groups

- Group A: none for default execution; the milestone is intentionally sequential because stable-root cleanup changes can affect shared freeze evidence.
- Group B: after `1394`, cards `1395`, `1396`, `1397`, `1398`, `1399`, and `1400` may be parallelized only when workers own disjoint files and no card changes shared specs or docs.

## Risk Register

- Risk: cleanup accidentally changes stable public API or visible response shape.
  Mitigation: require card-local tests, API inventory review, and symbol-change protocol before any exported change.
- Risk: mechanical file splits mix with behavior edits and become hard to review.
  Mitigation: keep store/log split cards mechanical, with no semantic changes unless a test reveals an existing bug.
- Risk: compatibility aliases are removed because they look stale.
  Mitigation: start with card `1394` and treat inventory decisions as the controlling source for the rest of the milestone.

- Card-level checks: run each card's target module race/test/vet commands plus docs or boundary checks listed in that card.
- Milestone-level checks: run the M-004 Acceptance Criteria commands after all cards are complete.
