# M-011: Benchmark Suite Publication

- **Branch:** `milestone/M-011-benchmark-suite`
- **Depends on:** M-008
- **Parallel OK:** yes

---

## Goal

Publish a reproducible HTTP benchmark suite that measures plumego router and middleware throughput, latency, and allocation profiles against `net/http` stdlib ServeMux, chi, and echo, with results embedded in the README.

---

## Architecture Decisions

- Benchmark files live in `internal/bench/`; they are `_test.go` files using `testing.B`, not a standalone binary.
- Compared targets: stdlib `net/http` ServeMux (Go 1.22+), chi v5, echo v4, plumego router. No other routers.
- Results table in `README.md` is generated by `go test -bench=. -benchmem ./internal/bench/...` and copy-pasted (no automated injection script).
- No external benchmark framework (e.g., wrk, vegeta) for the first publication — pure `testing.B` to keep the CI gate simple.
- chi and echo are added to `internal/bench/go.mod` as a **separate Go module** to keep the main module dependency-free.
- Do not add chi or echo to the root `go.mod`.
