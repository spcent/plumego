# Plan for M-007: Extension v1 Baseline Evidence Intake

Milestone: `M-007`
Objective: Use the published `v1.0.0` tag as the first real release evidence
point for remaining extension candidates, generate or register bounded baseline
artifacts, and keep all incomplete beta-promotion blockers explicit.
Constraints: no runtime behavior changes, no extension promotion, no owner
sign-off without an explicit owner record, and one commit per completed card.
Affected Modules: extension evidence, release, tasks.

## Phase Map

- Phase 1: Write the M-007 control plane and refresh stale release evidence
  guidance.
- Phase 2: Record first release-ref evidence for candidates that already had
  current-head snapshots.
- Phase 3: Add missing baseline snapshots for selected surfaces that still had
  no checked-in API snapshot.
- Phase 4: Reconcile active and blocked queues so the next blocker is visible
  without appearing executable.

## Risk Register

- Risk: one release ref is mistaken for complete release history.
  Mitigation: preserve `release_history_missing` until two release refs exist.
- Risk: current-head snapshots are counted as release-backed evidence.
  Mitigation: record v1 intake separately in docs when adding a ledger snapshot
  would clear `api_snapshot_missing` too early.
- Risk: owner sign-off is inferred from prior docs.
  Mitigation: keep `owner_signoff_missing` until an explicit owner record is
  added.

## Verification Strategy

- Card-level checks: run the ledger and maturity checks after every evidence
  update.
- Snapshot checks: compare generated snapshots to themselves to prove the
  snapshot parser accepts the artifact.
- Queue checks: run `agent-workflow` before committing task-card movement.
