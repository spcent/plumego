# Plan for M-019: Module Ecosystem Foundation

Milestone: `M-019`
Objective: Establish the foundational contract and tooling for community-authored
plumego extensions: a machine-readable community extension schema in
specs/community-extension.schema.yaml, a `plumego add` command that validates
schema compliance before modifying go.mod, and docs/EXTENSION_AUTHORING.md that
gives third-party developers a clear, checkable contract.
Constraints: no hosted extension registry, no auto-wiring or global registration,
no stable root package modifications, schema validation exits non-zero on any
violation before touching go.mod.
Affected Modules: specs, cmd/plumego, docs.

## Phase Map

- Phase 1: Orient — read specs/module-manifest.schema.yaml to understand the
  existing schema pattern; read cmd/plumego/commands/ to understand CLI wiring.
- Phase 2: Implement (parallel) — write the schema, add command, and authoring
  guide concurrently since they are independent artifacts.
- Phase 3: Test — write add command tests and internal/checks validation for all
  x/* module.yaml files against the new schema.
- Phase 4: Validate and Ship — run acceptance criteria, commit.

## Parallel Groups

- Group A: card 1590 — must complete first; defines the schema that both 1591 and
  1592 reference.
- Group B (parallel after A): cards 1591 and 1592 — independent consumers of the
  schema; no file overlap.
- Group C (sequential after B): internal/checks test for all x/* module.yaml files
  against the new schema.

## Risk Register

- Risk: the community extension schema is too strict and rejects valid x/* packages
  that follow current conventions.
  Mitigation: after writing 1590, run a dry-validation pass against all x/*/module.yaml
  files before committing the schema; adjust rules to match proven conventions.
- Risk: `plumego add` invokes `go get` which has network access side effects in CI.
  Mitigation: add command tests use a mock go get stub; real network access only in
  manual integration testing documented in M-019.verify.md.

## Verification Strategy

- Card-level checks: 1591 tests cover valid schema, missing field, forbidden import,
  and parse error paths; `go test ./cmd/plumego/...` must exit 0.
- Schema coverage check: internal/checks test runs all x/*/module.yaml files through
  the community-extension schema and reports any gaps.
- Docs accuracy check: docs/EXTENSION_AUTHORING.md x/rpc example paths are verified
  to exist in the repository.

## Exit Condition

- all three cards completed
- specs/community-extension.schema.yaml is machine-readable and all existing x/* packages pass it
- `plumego add` validates schema and exits non-zero on violations before touching go.mod
- docs/EXTENSION_AUTHORING.md exists with x/rpc worked example
- verify report shows pass
- milestone acceptance criteria ready for PR packaging
