### 3. SQLite 实现位置

- `internal/dbmanager/sqlite/inspector.go` — Inspector 接口实现
- `internal/handler/sqlite.go` — Upload（magic 校验 + 随机文件名）/ Download（magic 重校验）
- `internal/dbmanager/manager.go` buildDSN — SQLite 直接使用 FilePath 作为 DSN
- Placeholder 差异：SQLite 使用 `?1`, `?2`，MySQL 使用 `?`

### 4. MySQL 实现位置

- `internal/dbmanager/mysql/inspector.go` — Inspector 接口实现
- `internal/dbmanager/manager.go` buildDSN — `user:pass@tcp(host:port)/db?charset=utf8mb4...`
- `rows.go` quoteIdent — MySQL 使用反引号
- `export.go` sqlQuoteString — MySQL 需要转义反斜杠

### 5. 连接管理方式

KVStore（嵌入式），密码用 AES-GCM 加密，`SavePassword=false` 时不落盘。连接池 `Manager` 内存缓存 `map[connID]*sql.DB`，同一连接复用 pool。

### 6. SQL 执行方式

- **query.go**：单语句限制，`classifySQL` 分类，危险操作需 `confirmDangerous=true`，SELECT 自动追加 `LIMIT 1000`
- **rows.go**：所有 CRUD 通过 `sql.DB.QueryContext/ExecContext`，参数化占位符

### 7. 导入导出实现

- **导出**：流式逐行（不全量物化），支持 CSV 和 SQL（含 CREATE TABLE DDL 选项），已有行数限制（`LIMIT maxExportRows`）
- **导入**：`splitSQL()` 按字符级解析（支持 `/* */` / `--` / `#`），逐语句执行，危险语句需确认

### 8. 错误响应结构

```json
{
  "error": {
    "type": "bad_request|forbidden_request|not_found|validation|internal",
    "message": "人类可读消息",
    "details": {
      "code": "READONLY_VIOLATION",
      "confirm_required": true,
      "reason": "...",
      "error": "原始错误"
    }
  }
}
```

### 9. 危险操作确认机制

- SQL Console：`classifySQL()` → 返回 400 + `confirm_required: true` → 前端弹框 → 携带 `confirm_dangerous: true` 重发
- Row Update/Delete：必须携带 `"confirm": true`
- DropTable：必须携带 `?confirm=true`
- Import SQL：逐语句扫描，发现危险语句返回 400

### 10. 只读模式（后端强制执行）

`write.go` `guardReadonly()` 被以下位置调用：`query.Execute`、`row.Create/Update/Delete`、`ddl.CreateTable/AlterTable/DropTable`、`import.Import`。返回 HTTP 403 + `code: "READONLY_VIOLATION"`。

### 11. Identifier Quote 安全性

**安全**。`quoteIdent()` MySQL 用反引号+内部反引号转义，SQLite 用双引号+双引号转义。所有表名、列名经此函数处理。`sanitizeIdentifier()` 进一步只保留 `[a-zA-Z0-9_]`。

### 12. Filter Value 参数化

**安全**。`buildFiltersWhere()` 所有操作符均通过 `args = append(args, value)` 传参，无字符串拼接。

### 13. UPDATE/DELETE 无 WHERE 风险

- 行 CRUD 路径：`buildWhereFromPK()` 强制要求非空 PKMap + `confirm: true`，安全
- SQL Console：`hasWhereClause()` 字符串匹配（已改进为去除块注释后检测）

### 14. SQLite FilePath 路径穿越

**安全**。Upload 用 `crypto/rand` 生成 16 字节 hex 文件名，`filepath.Join(UploadDir, destName)` 写入服务器控制目录。Download 重校验 magic header。

### 15. 导入 SQL 危险确认

已改进：`splitSQL()` 现支持 `/* */` 块注释，危险语句检测调用 `classifySQL()`。

### 16. 密码日志泄露

**未发现**。`List()` 返回前密码清零，handler 层只打印 `connID`，DSN 字符串不被 log。`redactDSN()` 辅助函数可在调试场景使用。

## 四、多数据源演进评估

### 连接模型

`Connection` 结构体已有 `Driver DriverType` 和 `Options` 字段。枚举值仅 `mysql`/`sqlite`，扩展成本低。

### 资源树

`DatabaseTree.tsx` 三层结构（Connection → Database → Table）与 SQL 模型绑定。Redis/MongoDB 可复用两层，ES 复用一层（Index 直接对应 Table）。树骨架和 TreeItem 组件可完全复用。

### Workbench

纯 `<Outlet />`，路由决定渲染哪个页面。只需新增路由和页面组件即可支持新数据源。

### API 兼容性

现有 API 路径 `/db/:db/tables/:table/rows` 是 SQL 专用。Redis/MongoDB/ES 需要各自路由组，但 handler 模式（`connections.Get()` → `guardReadonly` → 执行 → `contract.WriteJSON`）可作为模板。

### Inspector 接口

`dbmanager.Inspector` 是 SQL 专用接口（`Databases/Tables/Columns/Indexes/ForeignKeys`）。不能直接复用到 Redis/ES，但模式可借鉴。
