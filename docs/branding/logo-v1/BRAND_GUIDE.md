# plumego logo-v1 brand guide (one page)

## Brand intent

plumego visual identity should communicate: standard library first, lightweight, composable, and engineering clarity.

## Approved logos

- Primary: `final/svg/plumego-primary-horizontal-light.svg`
- Primary dark background: `final/svg/plumego-primary-horizontal-dark.svg`
- Backup: `final/svg/plumego-backup-square-light.svg`
- Backup dark background: `final/svg/plumego-backup-square-dark.svg`
- Favicon package: `final/favicon/favicon.ico` and `final/favicon/favicon-32.png`

## Color tokens

### Monogram (P + Plum)
- `go_blue`: `#00ADD8`
- `slate`: `#1F2937`
- `white`: `#FFFFFF`

### Toolkit Emblem (Node + Route)
- `teal`: `#0F766E`
- `ink`: `#111827`
- `mist`: `#E5E7EB`

### Wordmark First (selected primary)
- `navy`: `#0B1F3A`
- `mint`: `#2DD4BF`
- `gray`: `#6B7280`

## Clear space and minimum size

- Clear space: at least `0.5x` around the mark (`x` = height of the lowercase `g` in wordmark).
- Minimum width:
  - Horizontal logo: `160px` digital
  - Square logo: `32px` icon
- Favicon recommendation: use square mark for `32px` and below.

## Usage rules

- Use `light` assets on white or very light backgrounds.
- Use `dark` assets on dark backgrounds.
- Use `bw` assets for single-color print or embossing.
- Keep aspect ratio unchanged.
- Do not add shadows, outlines, or gradients to logo masters.
- Do not recolor outside the defined palettes.

## Evaluation summary

Tested in:
- README hero area
- document cover mock
- GitHub avatar at `128px`
- favicon at `32px`

Result:
- Primary: Wordmark First
- Backup: Monogram
- Toolkit Emblem remains an alternative for architecture or product diagram contexts.
