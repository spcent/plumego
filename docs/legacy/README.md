# Legacy Docs

This directory holds historical migration markers, superseded reports, and phase summaries that are kept only for archival reference.

## Rules

- do not treat files under `docs/legacy/` as canonical product guidance
- prefer `README.md`, `README_CN.md`, `docs/modules/*`, `docs/architecture/*`, and active execution docs under `docs/other/*`
- move completed reports and obsolete migration markers here when they no longer guide current implementation

## Current Contents

- `modules/` - archived module migration markers
- `reports/` - superseded readiness reports and phase summaries
- `notes/` - superseded plans, audits, and one-off standards snapshots
- `topics/` - historical domain plans, deep dives, and implementation writeups
- `api/` - archived ad hoc API reference pages
- `proposals/` - historical roadmap or redesign proposals
