# Plumego v1.1.0

## Summary

Plumego v1.1.0 is a post-v1 minor release focused on release-control-plane hardening, agent workflow discoverability, website documentation parity, and extension evidence readiness. It does not intentionally change the stable-root public API surface.

## Status

- Final decision: GO on May 18, 2026.
- Final tag: pending card 1503.
- Gate command: `GOCACHE=/private/tmp/plumego-gocache make gates`.
- Gate result: PASS, recorded in the Gate Run section below.
- Go toolchain: `go version go1.26.0 darwin/arm64`.

## Evidence Reviewed

- Local `make gates` passed on May 18, 2026.
- Stable API snapshot check in the gate output reported `snapshots match`.
- Extension maturity and beta-evidence checks completed in the gate output.
- M-006 maintenance evidence is recorded in `docs/release/POST_V1_EVIDENCE.md` and `tasks/milestones/done/M-006.md`.
- M-007 extension v1 baseline intake is recorded in `tasks/milestones/done/M-007.md` and `specs/extension-beta-evidence.yaml`.

## What's New

- Agent workflow improvements: route, diff-validation, task-bundle, recipe, and manifest guidance were expanded so agents can find landing zones and validation paths more deterministically.
- Website and docs updates: release posture, status, architecture, migration, benchmark, and bilingual docs pages were synchronized and rebuilt under the website gate.
- Extension evidence intake: selected experimental extension candidates now have their first `v1.0.0` release-ref evidence point and baseline snapshots where applicable.
- Queue hygiene: active, blocked, and done card lifecycle rules were tightened so near-term work remains executable and archived cards retain validation outcomes.
- Maintenance hardening: GitHub Actions runtime and CLI install guidance were aligned with the source-checkout validation model for the nested CLI module.

## Known Issues

- Tagged `go install github.com/spcent/plumego/cmd/plumego@<tag>` remains unadvertised because `cmd/plumego` is a nested Go module; CLI validation is source-checkout based.
- Several extension promotion candidates still have explicit evidence blockers in `specs/extension-beta-evidence.yaml`: `release_history_missing`, `api_snapshot_missing`, and `owner_signoff_missing`.
- No runtime regression is recorded in the v1.1.0 gate evidence.

## Upgrade Path

Applications using the stable roots can upgrade from `v1.0.0` to `v1.1.0` without code changes expected from stable-root API drift. Re-run local application tests, then run the relevant Plumego gates for any extension packages used directly.

Do not treat first-release-ref extension evidence as beta status. Keep experimental extension usage pinned to documented behavior and review each module primer before relying on compatibility guarantees.
