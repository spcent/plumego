# Plumego v1.0.0-rc.1

## Status

- Main module: release candidate for v1.
- Stable roots: GA candidate surface for `core`, `router`, `contract`, `middleware`, `security`, `store`, `health`, `log`, and `metrics`.
- Canonical reference app: supported reference aligned with stable-root usage.
- CLI: v1 hardening scope, supported as a command-line tool and not as a Go import surface.
- Extensions: experimental unless explicitly listed otherwise. No `x/*` module is promoted to beta or GA in this release candidate.

## Included

- CI and local release gates now use the same live control-plane paths.
- Local `make gates` now includes extension maturity checks, beta-evidence checks, stable-root coverage, `cmd/plumego` submodule checks, and website generated-data validation.
- Local `make gates` now includes the deprecation inventory check for registered compatibility aliases, unsupported-operation placeholders, and TODO guardrails.
- Website release and roadmap generated data is synchronized from repository source documents.
- Milestone verification artifacts record M-001 trust baseline, M-002 stable-root freeze evidence, and M-003 extension evidence blockers.

## Compatibility Decisions

- `middleware/httpmetrics.Observer` is retained for v1 as the app-facing alias for `metrics.HTTPObserver`; this keeps HTTP metrics middleware wiring local to the middleware package.
- `reference/workerfleet` is not part of the canonical v1 reference surface. Its unimplemented worker registration path remains a non-blocking reference placeholder.
- Experimental `x/*` compatibility aliases are retained but not promoted to stable API. This includes `x/gateway` gateway-prefixed entrypoint aliases, `x/messaging` family entrypoint aliases, `x/mq` pubsub re-exports, `x/data/idempotency` stable-contract re-exports, and `x/ai/multimodal` provider type aliases.
- Experimental `x/*` compatibility helpers remain non-stable. `x/ipc.RemoteAddrString` is retained as a migration helper; typed `Client.RemoteAddr().String()` remains preferred.
- Experimental `x/mq` MQTT and AMQP bridge entrypoints remain explicit unsupported operations for v1. `Config.Validate` fails closed when those bridges are enabled.
- Experimental `x/mq` ack and trie defaults remain compatibility-preserving opt-in behavior for v1.

cd cmd/plumego && go test -race -timeout 60s ./...
cd cmd/plumego && go test -timeout 20s ./...
cd website && pnpm sync
cd website && pnpm check

does not leave uncommitted generated-file changes.
